Basics

A git repository consists of three areas. The working directory, staging area, and the .git directory

Working Directory

This is the area in your filesystem where you modify existing files and add new ones

File state

Files in the working directory and staging area have four states: untracked, unmodified, modified, and finally staged

Untracked

Any new files added to the working directory are considered untracked.

Unmodified

Files in the working area that haven't been changed since the last commit or clone are considered unmodifed

Modified

Files that have been previously commited or were part of the initial clone - and then subsequently changed are considered modified

Staged

Untracked or Modified can be set to the staged state by doing a git add command.

Staging Area

When you're at a point where you can commit your work. You use the command git add to add the files you wish to commit to the stage

.git Directory

After adding files to the staging area, you can commit those files to the .git directory by issuing a git commit command.

This snapshots the commited files, creates an reference to the commit by check-summing them, and pushes that into the local repository.
