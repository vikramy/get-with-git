GitHub

GitHub is a web layer on top of git that adds a lot of collaborative features.

README.md

Projects on GitHub typically have a readme.md file in their root directory. It's in markdown format and is shown on the repository's landing page. It typically has a description of and basic instructions for the project.

Let's have a look

Wiki

GitHub repositories have the option to have a wiki.

For internal projects, the wiki can be a great place to document design decisions or APIs.

For open source projects, wikis are often used for more detailed documentation.

The wiki is a git repository, so it can be cloned and edited locally.

Let's have a look

Issues

GitHub has issue tracking built in. Can be used for bugs, enhancement requests, etc. Issue threads can be assigned, labelled, and tied to milestones

Let's have a look

Pull requests

The typical workflow on projects is for members to fork the main repository (upstream) into their own repository (origin).

Once their work is ready to merge into upstream, they'll create a pull request. This will give other team members the opportunity to do a code review at this point.

Team members can comment on individual lines of the pull request.
