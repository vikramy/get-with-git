Branching

Structure of a commit

In order to understand branching in git, it's helpful to understand what an individual commit is.

Commit object

The commit object contains a commit comment, author, and a pointer to the commit tree. The commit object is referened by the checksum of the snapshot of the files

Commit objects also contain a reference to the checksum of the commit before it

Tree

The tree contains a list of all the directories and files contained in the commit

Blobs

Git refers to the version of a file as a blob

A git Branch

A branch in git is simply a pointer to a particular commit.

Creating a branch in git is considered "cheap" in git, since it only involves creating a new pointer

If you've checked out a particular branch, there is also a HEAD pointer pointing to the branches pointer
