Step 4
Clean Slate

In the previous step we modified one file, added one file, and then staged those changes to be committed.

We're going to undo all of that.

  1. git reset HEAD public/app/practice/final.html
  2. git status

Now were back to the state where final.html is modified, but no longer staged and ready to be committed.

git checkout public/app/practice/final.html

All changes to final.html are gone and lost forever.

  1. git reset HEAD .
  2. git status

Just like git add ., the '.' will reset all files and remove them from the Staging Area.
