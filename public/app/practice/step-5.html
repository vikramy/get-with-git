Make Changes in the Working Directory Redux

Once again, open public/app/practice/final.html and add your information into the empty row (still ignore favorite sandwich).

Panic!

You've been informed to drop what you're doing and do a hotfix. The client no longer needs to know your favorite sandwich.

  1. git stash
  2. git branch hotfix
  3. git checkout hotfix

Your changes are now in the stash stack.

Open public/app/practice/final.html and remove the <th>favorite sandwich</th> tag.

  1. git add public/app/practice/final.html
  2. git commit -m "removed unnecessary th"

You've just added the file to stage, and then committed it.

  1. git checkout master
  2. git merge hotfix
  3. git stash pop

The hotfix is done and merged into master. You can now go back to work getting your information into the repository.

git status

You'll see that public/app/practice/final.html is listed as Changes not staged to commit - modified.

Use the shortcut to commit all modified files (note! untracked files will not be added / commited)

git commit -am 'added my personal info'
