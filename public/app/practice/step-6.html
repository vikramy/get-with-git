Dealing with Conflict

Your personal information is now added and committed to your local repository.

Before committing you should always update your local branch with the remote repository.

git pull upstream master

Before committing you should always update your local branch with the remote repository.

git status

You'll see that someone else has made changes to their personal information in public/app/practice/final.html and it's causing a conflict

Open the file to view the conflict

<<<<<<< HEAD:public/app/practice/final.html
            <td>Your Name</td>
            <td>yourusername</td>
            <td>your.email@icfi.com</td>
=======
            <td>Jeff Heinrichs</td>
            <td>icf-jheinrichs</td>
            <td>jeff.heinrichs@icfi.com</td>
>>>>>>> [some checksum value]:public/app/practice/final.html

Resolve the conflict by moving your information to the to of the list and deleting the extra conflict marker code

git commit -m 'merged upstream master into my master'

Undoing merges

If at first glance a merge seems particularily messy, you can back out of the merge by issuing the following:

git merge --abort
