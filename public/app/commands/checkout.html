Git Checkout

git checkout [branch name]

Switch from current branch to branch specified. Git will allow you to switch to another branch with modified files. They will remain modified. You can then inadvertently commit the changes to the wrong branch. Either be sure to commit files before checking out another branch, or stash your changes.

git checkout -b [branch name]

This is a shortcut to create a new branch and switch to it.

git checkout [file name]

Allows you to remove any changes from a modified file and restore it to an unmodified state. Be careful with this command - you will lose work if you checkout the wrong file (this is the equivalent of SVN revert). If a file is staged, this will have no effect.
